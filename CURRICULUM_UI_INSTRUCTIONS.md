#### Component Behaviour

**Week accordion** (use `Accordion` from `ui/accordion.tsx`):
- One `AccordionItem` per week
- Header shows: `{label} — {theme}` + `{concepts.length} concepts · {total_problems} problems`
- Refresher week shows a "Prerequisites" badge (use `Badge` from `ui/badge.tsx`)
- Default open: `week1`

**Concept card** (use `Card` from `ui/card.tsx`):
- Title: concept name
- "Introduced in" row:
  - Shows `{code}` and `{timestamp}`
  - `[▶ Learn]` button — opens `introduced_in.deep_link` in a new tab
  - Small text: `{examples_count} examples in lecture`
  - If `introduced_in` is null, show "Introduction not found"
- "Practiced in" row (only if `practiced_in.length > 0`):
  - Shows tutorial codes as badges: `W1_T1`, `W1_T2`
  - Shows total problems count
  - Expandable problem list: each problem shows `summary` + `[▶]` button linking to `deep_link`
- `is_prerequisite: true` → show a "Prerequisite" badge on the card

**Problem list expansion:**
- Default collapsed (show only count)
- Click "Show problems" → expand inline list
- Each problem item: `summary` text + small `[▶ {timestamp}]` link button

**All `[▶]` / `[Watch]` links:**
- `target="_blank" rel="noopener noreferrer"`
- Use `deep_link` field directly — it already contains `&t=<seconds>`

### 3. Wire into App.tsx

The app uses `viewMode` state (`"welcome" | "transcript" | "search"`).

Add `"curriculum"` to the union and render `<CurriculumPage />` when active.

Add a **"Course Map"** nav button in the top bar (next to the existing dark mode toggle). Use a `Map` or `BookOpen` icon from `lucide-react`.

Clicking "Course Map" sets `viewMode` to `"curriculum"`.
Clicking it again (or the back button) returns to `"welcome"`.

Do **not** change any existing view modes or break the search/transcript flow.

## Implementation Notes

- Import curriculum data at the top of `CurriculumPage.tsx`:
  ```ts
  import curriculumData from "@/data/curriculum.json";
  import type { CurriculumData } from "@/types";
  const curriculum = curriculumData as CurriculumData;
  ```
- The `@/` alias resolves to `src/` — see `vite.config.ts`
- Use `cn()` from `@/lib/utils` for conditional class names (already used everywhere)
- Use TailwindCSS v4 utility classes — same pattern as existing components
- Dark mode: the app toggles `dark` class on `<html>` — use `dark:` variants
- Do not add any new npm dependencies
